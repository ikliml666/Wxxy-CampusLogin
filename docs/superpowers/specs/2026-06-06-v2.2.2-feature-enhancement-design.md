本文档涵盖三个独立功能目标的详细设计：

## 目标1：中英语言切换

### 技术选型

- **i18n库**：react-i18next（基于i18next）
- **翻译格式**：集中式JSON，按模块分namespace
- **默认语言**：中文（zh）
- **支持语言**：中文（zh）、英文（en）

### 文件结构

```
frontend/src/
├── i18n/
│   ├── index.ts              # i18next初始化配置
│   ├── locales/
│   │   ├── zh.json           # 中文翻译
│   │   └── en.json           # 英文翻译
│   └── types.ts              # 翻译key类型定义（可选）
```

### 语言切换机制

1. **初始化**：`i18next.init()` 检测优先级：localStorage `app-language` > navigator.language > 默认 `zh`
2. **切换**：`i18next.changeLanguage(lang)` + 持久化到 localStorage
3. **同步后端**：语言变更时通过 Tauri IPC 通知后端
4. **Zustand集成**：在 `useAppStore` 中添加 `language` 状态，与 i18next 双向同步

### UI变更

- **TitleBar**：在主题切换按钮旁添加语言切换图标按钮（地球图标），点击切换中/英，tooltip显示当前语言
- **OnboardingWizard**：步骤0（欢迎页）添加语言切换按钮，位于底部"跳过/下一步"按钮上方
- **语言切换无动画**：切换后立即生效，不重新加载页面

## 目标2：老旧日志自动删除

### 后端改动（Rust）

**修改文件**：`src-tauri/src/infra/logger.rs`

1. **新增配置项**：在 `AppConfig` 中添加 `log_retention_days: u32` 字段（0=永久保存，默认7）
2. **定时清理任务**：在 `logger-worker` 线程中增加定时检查逻辑
   - 每 1 小时检查一次日志文件修改时间
   - 删除超过 `log_retention_days` 天的日志文件
   - `log_retention_days = 0` 时跳过清理
3. **配置变更响应**：前端修改保存时间后，通过 IPC 命令通知后端更新配置并立即触发一次清理

**新增 IPC 命令**（`src-tauri/src/commands/system.rs`）：
- `set_log_retention_days(days: u32)` - 设置日志保存天数
- `get_log_retention_days() -> u32` - 获取当前设置

### 前端改动

**修改文件**：`frontend/src/shared/LogPanel.tsx`

在日志面板顶部工具栏添加"保存时间"下拉选择器：
- 选项：3天 / 7天（默认）/ 14天 / 30天 / 永久保存
- 位于"行数选择器"和"清空按钮"之间
- 使用 Radix UI Select 组件
- 选择后立即保存到后端配置

### 数据流

```
用户选择保存时间 → 前端调用 set_log_retention_days IPC
    → 后端更新 AppConfig.log_retention_days
    → 后端立即执行一次清理检查
    → 后端定时任务每小时检查一次
    → 删除超过保存时间的日志文件
```

### 清理逻辑

```rust
fn cleanup_old_logs(log_dir: &Path, retention_days: u32) {
    if retention_days == 0 { return; } // 永久保存
    let cutoff = SystemTime::now() - Duration::from_secs(retention_days as u64 * 86400);
    for entry in fs::read_dir(log_dir).unwrap() {
        let entry = entry.unwrap();
        if let Ok(metadata) = entry.metadata() {
            if let Ok(modified) = metadata.modified() {
                if modified < cutoff {
                    let _ = fs::remove_file(entry.path());
                }
            }
        }
    }
}
```

## 目标3：CSS动画GPU加速优化

### 问题分析

当前GPU进程CPU占用5%而GPU占用<5%，核心原因：
- 部分CSS动画使用了 `opacity` + `background`/`box-shadow` 等非合成属性，触发重绘
- `FluidBackground` 的渐变层使用 `background-position` 动画，这是重绘级操作
- 循环动画持续运行，即使不可见也可能消耗资源

### 优化策略：扩大GSAP迁移

将所有循环动画和关键视觉动画迁移到GSAP，保留CSS仅用于简单的一次性过渡。

#### 迁移到GSAP的动画（12个）

| 动画名 | 类型 | 迁移方式 |
|--------|------|---------|
| `statusBreathe` | 呼吸灯 | GSAP `gsap.to()` 循环，`force3D:true` |
| `emptyBreathe` | 呼吸灯 | GSAP `gsap.to()` 循环 |
| `capsuleBreathe` | 呼吸灯 | GSAP `gsap.to()` 循环 |
| `borderGlowRed` | 发光脉冲 | 伪元素 + `transform:scale()` + `opacity` 模拟发光 |
| `capsuleGlowPulse` | 发光脉冲 | 伪元素 + `transform:scale()` + `opacity` |
| `signalGlowPulse` | 发光脉冲 | 伪元素 + `transform:scale()` + `opacity` |
| `fluidGradientShift` | 渐变移动 | `transform: translate3d()` 移动渐变层 |
| `fluidOrb1Move` | 光球移动 | GSAP `gsap.to()` + `force3D:true` |
| `fluidOrb2Move` | 光球移动 | GSAP `gsap.to()` + `force3D:true` |
| `heartbeat` | 脉冲 | GSAP `gsap.to()` 循环 |
| `statusPulse` | 脉冲 | GSAP `gsap.to()` 循环 |
| `loadingPulseScale` | 加载脉冲 | GSAP `gsap.to()` 循环 |

#### 保持CSS的动画（~12个）

| 动画名 | 理由 |
|--------|------|
| `cardEnter`/`staggerSlideUp`/`windowReveal` | 一次性入场，已用transform+opacity |
| `dialogOverlayIn/Out`/`dialogContentIn/Out` | Framer Motion管理 |
| `rippleExpand`/`iconSpinIn`/`refreshShake`等 | 一次性微交互，性能影响小 |
| `logFlash`/`numberFlash`/`checkFlash` | 一次性闪烁 |
| `barElastic`/`signalBarEnter` | 一次性弹性动画 |

#### CSS层优化（对所有保留的CSS动画）

- 添加 `will-change: transform` 或 `will-change: opacity` 提示
- 添加 `contain: paint` 隔离重绘范围
- 确保只操作 `transform` + `opacity` 合成属性

### GSAP迁移收益

- `force3D: true` 自动添加 `translateZ(0)` 强制GPU合成层
- `autoSleep: 60` 空闲时自动暂停循环动画
- `will-change` 自动管理，动画结束后移除
- `box-shadow` 动画改为 transform 模拟，消除重绘
- 统一管理所有循环动画生命周期

### 预期效果

- GPU进程CPU占用从5%降至1-2%
- GPU占用提升至10-20%（正常GPU加速水平）
- 动画流畅度提升（减少主线程阻塞）
- 视觉表现不变

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| i18n翻译遗漏 | 部分文本未翻译 | 编写脚本扫描硬编码中文，逐一校验 |
| 日志清理误删 | 用户重要日志被删 | 默认7天+永久保存选项+清理前日志 |
| GSAP迁移回归 | 动画视觉效果变化 | 逐个迁移+视觉对比验证 |
| i18n包体积 | react-i18next增加约30KB | 按namespace懒加载翻译文件 |
